# EPIC-00 Implementation Plan — Project Foundations

> **Issue:** [#1 — [EPIC-00] Project foundations](https://github.com/Joncallim/macdown_2/issues/1)
> **Product:** MacDown 2 · Bundle ID: `com.joncallim.macdown2` · App target: `MacDown2` · CLI target: `macdown2`
> **Branch:** work on `epic/00-foundations`, PR into `rewrite/main`
> **Environment:** a Mac with Xcode 26 and Homebrew. **Estimated effort: ½–1 day.**
> **Result:** an empty-window MacDown 2 app that builds locally and in CI, a SwiftPM
> module skeleton, lint/format tooling, and contributor docs.

## Ground rules (binding)

1. macOS **26.0** deployment target everywhere. Never write an availability check.
2. Swift 6 language mode + strict concurrency **on from day one**.
3. SPM only. No CocoaPods, no git submodules, no Carthage, ever.
4. Do not modify the legacy tree (`MacDown/`, `Dependency/`, `macdown-cmd/`, `Tools/`).
5. All new code lives under `MacDown2/` at repo root.
6. `MacDown2.xcodeproj` and `MacDown2/Info.plist` are **generated by XcodeGen** and
   gitignored. Never edit the project by hand; edit `project.yml` and regenerate.

## Step 2 — SwiftPM package skeleton (`MacDown2/Packages/MacDownKit`)

Create the directory layout:

```
MacDown2/
├── MacDown2/                 # app target sources   (Step 3)
├── MacDown2CLI/              # CLI target sources   (Step 3)
├── Packages/
│   └── MacDownKit/
│       ├── Package.swift
│       ├── Sources/<Module>/<Module>.swift          (11 modules)
│       └── Tests/<Module>Tests/<Module>Tests.swift  (11 test targets)
└── project.yml               # (Step 4)
```

Generate the 11 module stubs and their tests:

```bash
mkdir -p MacDown2/Packages/MacDownKit && cd MacDown2/Packages/MacDownKit
modules=(FileCore AppSettings Themes Workspace FileTree EditorCore Highlighting MarkdownEngine Preview OutlineUI ExportService)
for m in "${modules[@]}"; do
  mkdir -p "Sources/$m" "Tests/${m}Tests"
  cat > "Sources/$m/$m.swift" <<EOF2
/// $m — module stub. See planning/epics/ and planning/MIGRATION_PLAN.md §4 for its role.
public enum $m {
    public static let moduleName = "$m"
}
EOF2
  cat > "Tests/${m}Tests/${m}Tests.swift" <<EOF2
import Testing
@testable import $m

@Test func moduleLoads() {
    #expect($m.moduleName == "$m")
}
EOF2
done
```

## Step 6 — CI (GitHub Actions)

`.github/workflows/ci.yml` (**repo root**, not inside `MacDown2/`):

```yaml
name: CI

on:
  push:
    branches: [rewrite/main]
    paths: ['MacDown2/**', '.github/workflows/ci.yml']
  pull_request:
    branches: [rewrite/main]
    paths: ['MacDown2/**', '.github/workflows/ci.yml']

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  lint:
    runs-on: macos-26
    steps:
      - uses: actions/checkout@v4
      - name: Install tools
        run: brew install swiftlint swiftformat
      - name: SwiftFormat (lint)
        run: swiftformat --lint MacDown2
      - name: SwiftLint (strict)
        run: swiftlint lint --strict MacDown2

  build-and-test:
    runs-on: macos-26
    steps:
      - uses: actions/checkout@v4
      - name: Select Xcode 26
        run: |
          XCODE=$(ls -d /Applications/Xcode*.app | grep "26" | head -1 || true)
          if [ -z "$XCODE" ]; then XCODE=$(ls -d /Applications/Xcode*.app | head -1); fi
          sudo xcode-select -s "$XCODE/Contents/Developer"
          xcodebuild -version
      - name: Install XcodeGen
        run: brew install xcodegen
      - name: Package build + tests
        working-directory: MacDown2/Packages/MacDownKit
        run: |
          swift build
          swift test
      - name: Generate Xcode project
        working-directory: MacDown2
        run: xcodegen generate
      - name: Build app + CLI
        run: |
          xcodebuild -project MacDown2/MacDown2.xcodeproj -scheme MacDown2 -destination 'platform=macOS' build
          xcodebuild -project MacDown2/MacDown2.xcodeproj -scheme macdown2 -destination 'platform=macOS' build
```

> Fallback: if the `macos-26` runner label is unavailable, use `macos-15`
> (arm64 images carry Xcode 26). The xcode-select step picks the right Xcode
> either way.

**Verify:** push the branch, open the PR, both jobs go green.

## Handoff notes / known pitfalls

- **`.macOS(.v26)` fallback** — see Step 2 note; do not lower the Xcode project's deployment target instead.
- **Never hand-edit the xcodeproj** — it is generated + gitignored; change `project.yml`.
- **Tests belong to the package**, not an Xcode test target, in E00. App-level
  XCUITests arrive with the document-lifecycle epics (E01/E03).
- **Legacy tree untouched** — the old `.travis.yml`, Podfile, pods, and submodule
  stay as-is on this branch; they are retired when `rewrite/main` becomes the product.
- If SwiftLint/SwiftFormat fight each other, SwiftFormat wins for layout; adjust
  `.swiftlint.yml` rather than weakening `.swiftformat`.
